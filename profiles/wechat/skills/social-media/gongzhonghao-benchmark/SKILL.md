# 公众号对标库管理

## 定位

用户提供一篇文章（链接或文本） → 按 5 个模块拆解 → 同时产出：
- **飞书表格追加一行**（总览索引，快速筛选对比）
- **本地 md 文件**（完整拆解，`~/.hermes/profiles/wechat/projects/gongzhonghao/对标文章/`）

## 目录结构

```
~/.hermes/profiles/wechat/projects/gongzhonghao/
├── 对标文章/           # 每篇一个 md
│   └── 对标-{账号简称}-{关键词}.md
├── 选题/
├── 已发布/
└── 模板/
    └── 对标文章模板.md
```

## 拆解规范

每篇文章拆 5 个模块，不偷懒不跳过：

### 1. 标题拆解
```markdown
## 1. 标题拆解
- **公式**: [结构公式，如：痛点+数字+低门槛+悬念]
- **逐句拆解**:

| 句式 | 作用 |
|------|------|
| "xxx" | 痛点共鸣 |
| "xxx" | 具体数字降怀疑 |

- **情绪节奏**: [焦虑→希望 / 好奇→共鸣 / ...]
- **可套用句式**: [填空题式，可直接套用]
```

### 2. 结构拆解
```markdown
## 2. 结构拆解
**整体框架**: [用箭头串联，如：认知反转 → 逻辑论证 → 三步操作 → CTA]

| 段落 | 功能 | 手法 |
|------|------|------|
| 导语 | 建立共情+认知反转 | 先站读者一边，再转折制造认知差 |
| 第一部分 | 降低门槛 | 连续排比否定 |
| 第二部分 | 预判质疑并主动解答 | 类比法消解顾虑 |
| 第三部分 | 降低行动阻力 | 极简步骤 |
| 结尾 | 情绪收束+推行动 | 金句+CTA |
```

### 3. 金句/素材
```markdown
## 3. 金句/素材
| 原文 | 可复用方式 |
|------|-----------|
| "xxx" | 适用于xxx场景 |
```

### 4. 转化方式
```markdown
## 4. 转化方式
- **引流路径**: [画箭头表示: 文章内容 → 过渡话术 → 扫码/加微信 → 私域承接]
- **CTA 手法**: [认知包装 / 福利名义 / 限时 / ...]
- **问题/可优化点**: [如果发现转化链路有断裂，指出]
```

### 5. 启发选题
```markdown
## 5. 启发选题
- [[选题-xxx]]
- [[选题-xxx]]
```
如果发现了明确的选题灵感，同步创建选题文件到 `../选题/` 目录，`[[ ]]` 双向链接。

## 飞书表格字段（13 列）

| # | 字段名 | 类型 | 说明 |
|---|--------|------|------|
| 1 | 序号 | 数字 | 自增 |
| 2 | 来源账号 | 文本 | 公众号名 |
| 3 | 文章标题 | 文本 | 原文标题 |
| 4 | 原文链接 | 链接 | 方便回看 |
| 5 | 文章类型 | 单选 | 干货 / 故事 / 推广 / 认知 |
| 6 | 可用度 | 单选 | ⭐⭐⭐ / ⭐⭐ / ⭐ |
| 7 | 标题公式 | 文本 | 痛点+数字+低门槛 |
| 8 | 标题情绪 | 文本 | 焦虑→希望 |
| 9 | 核心结构 | 文本 | 认知反转→逻辑论证→三步操作→CTA |
| 10 | 核心钩子 | 文本 | 开头用什么手法勾住读者 |
| 11 | 转化手法 | 文本 | 引流/加微信/付费的方式 |
| 12 | 启发选题 | 文本 | [[选题-xxx]] |
| 13 | 拆解日期 | 日期 | YYYY-MM-DD |

## 标签体系

三套标签，全库统一：

**文章类型**: `#干货` `#故事` `#推广` `#认知`

**结构特征**: `#认知反转开头` `#排比否定降门槛` `#三步操作` `#案例驱动` `#预判质疑` `#金句收尾`

**可用度**: 写在 frontmatter `rating` 字段

## 工作流

1. 用户发送文章（文本或链接）
2. 逐模块拆解（标题 → 结构 → 金句 → 转化 → 选题）
3. 新建本地 md 文件到 `对标文章/` 目录
4. 有启发选题时，同步创建选题文件，`[[ ]]` 双向链接
5. 完成后向用户展示拆解结果，确认后追加飞书表格

## 飞书表格连接

首次使用时，需要用户提供飞书表格的 URL 或文档 ID。之后的拆解自动追加到该表格。

## Obsidian 迁移

- 整个 `gongzhonghao/` 文件夹拖入 Obsidian Vault
- frontmatter 完整，可直接搜索/过滤
- `[[ ]]` 双向链接在 Obsidian 图谱中自动关联
- 标签体系统一，跨文件可检索

## 注意事项

- **文件名规范**: `对标-{账号简称}-{关键词}.md`，账号简称 2-4 字，关键词 2-5 字
- **不跳过模块**: 5 个模块全部拆解，即使某个模块信息不足也要标注"未观察到"
- **标题必须拆到可套用句式**: 不只是总结公式，要给出填空题式模板
- **启发选题用 [[ ]]**: 保证 Obsidian 迁移后双向链接生效
